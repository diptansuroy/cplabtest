Pipe & Redirection
# Pipe: ls output → grep filter
ls -l | grep ".txt"

# Output redirect (overwrites)
wc -l test.c > result.txt

# Append (does NOT overwrite)
echo "hello" >> result.txt

# Chain: count .c files
ls | grep ".c" | wc -l
KEYwc -l test.c shows filename + count. wc -l < test.c shows count only. > overwrites; >> appends. Pipe | connects stdout → stdin.

06
Functions & Scope
pass-by-value · scope · recursion · function declarations · blocks
CFunctionsRecursion
Function Pattern
/* Declaration */
int factorial(int n);

/* Recursive definition */
int factorial(int n) {
    if(n <= 1) return 1;
    return n * factorial(n-1);
}

/* Fibonacci recursive */
int fib(int n) {
    if(n<=1) return n;
    return fib(n-1) + fib(n-2);
}
Scope & Pass-by-Value
  • Pass-by-value: function gets a copy — changes don't affect the original in caller
  • Block scope: variables in { } live only within that block
  • Uninitialized locals have garbage values
  • Swap via pass-by-value does NOT work in caller
07
Arrays
linear search · binary search · selection sort · pass-by-reference
CArraysSorting
KEYArrays are passed by reference — changes inside function affect the original. sizeof(arr) inside a function = 8 (pointer size), NOT array size. Always pass size as a parameter.
Linear & Binary Search
/* Linear O(n) — any array */
int linSearch(int a[], int n, int x) {
    for(int i=0; i<n; i++)
        if(a[i]==x) return i+1;
    return -1;
}

/* Binary O(log n) — SORTED ONLY */
int binSearch(int a[], int n, int x) {
    int lo=0, hi=n-1;
    while(lo<=hi) {
        int mid=(lo+hi)/2;
        if(a[mid]==x) return mid+1;
        else if(a[mid]<x) lo=mid+1;
        else hi=mid-1;
    }
    return -1;
}
Selection Sort (Decreasing)
void selSortDec(int a[], int n) {
    for(int i=0; i<n-1; i++) {
        int mx = i;
        for(int j=i+1; j<n; j++)
            if(a[j]>a[mx]) mx=j;
        int t=a[i]; a[i]=a[mx]; a[mx]=t;
    }
}
/* Print ascending: loop backwards */
for(int i=n-1; i>=0; i--)
    printf("%d ", a[i]);

Multi-file C Projects
header files · .o object files · linking · shell scripts · linker errors
C.h filesShell ScriptExam!
Project Workflow
# Compile each .c to .o (no link)
gcc -c func.c
gcc -c main.c

# Link .o files → executable
gcc -o myexe func.o main.o

# Run
./myexe

# Shell script (myScript.sh)
rm -f *.o *exe
gcc -c func.c
gcc -c main.c
gcc -o myexe func.o main.o
./myexe

# Run script
sh myScript.sh
Linker Errors
  • Undefined reference to 'foo': forgot to link .o file containing foo()'s definition
  • Multiple definition of 'foo': two .o files both define the same function
  • Undefined reference to main: main .o not included, or typo Main() vs main()
  • Only ONE main() allowed across all linked files
  • .h = declarations only; .c = implementations

CPointersExam!
Pointer Basics & Arithmetic
int x = 10;
int *ptr = &x;    // ptr = address of x
printf("%d", *ptr);  // 10 (dereference)
*ptr = 20;          // x becomes 20!

/* Pointer arithmetic on arrays */
int arr[] = {22,54,78};
int *p = arr;
*(p++)  // 22, then p moves to arr[1]
arr[1] == *(arr+1) == *(p+1)
Call-by-Reference
/* Return multiple values */
void circleData(double r,
    double *area, double *peri) {
    *area = 3.14159 * r * r;
    *peri = 2 * 3.14159 * r;
}
/* Call: */
double a, p;
circleData(5.0, &a, &p);

/* Arrays as pointers (same!) */
void foo(int arr[])  { }
void foo(int *arr)   { }  // identical

Dynamic Memory & CLI Args
malloc · calloc · free · argc · argv · atoi · heap vs stack
mallocfreeargc/argvExam!
Dynamic Allocation
#include <stdlib.h>

/* malloc: uninitialized */
int *arr = (int*)malloc(n * sizeof(int));

/* calloc: zero-initialized */
int *arr = (int*)calloc(n, sizeof(int));

/* Always check NULL */
if(arr == NULL) { /* handle error */ }

arr[0] = 10;  // use like normal array

/* ALWAYS free heap memory */
free(arr);
Command Line Arguments
int main(int argc, char *argv[]) {
    // argc = count (includes prog name)
    // argv[0] = program name "./a.out"
    // argv[1] = first arg (string)
    
    int n = atoi(argv[1]); // str→int
}
/* ./prog 42 hello → argc=3 */
/* argv[1]="42", argv[2]="hello" */
WARNHeap memory is NOT auto-released. Always free(ptr). Never dereference uninitialized pointer: int *p; *p = 5; is undefined behavior.

Right Rotate Array by K Positions
arraysmoduloin-place
EASY
▼Show Solution
Problem
Solution
Key Insight
Implement void rotateArray(int arr[], int n, int rotateBy) that rotates the array to the right by rotateBy positions in-place.
arr=[1,2,3,4,5], rotateBy=2 → [4,5,1,2,3] arr=[10,20,30,40], rotateBy=18 → [30,40,10,20]
Hint: Normalize k = rotateBy % n. Use the three-reversal trick: reverse [0..n-k-1], reverse [n-k..n-1], reverse whole.
f1.c
void reverse(int arr[], int lo, int hi) { while(lo < hi) { int tmp = arr[lo]; arr[lo++] = arr[hi]; arr[hi--] = tmp; } } void rotateArray(int arr[], int n, int rotateBy) { rotateBy = rotateBy % n; if(rotateBy == 0) return; reverse(arr, 0, n-rotateBy-1); reverse(arr, n-rotateBy, n-1); reverse(arr, 0, n-1); }
Key Insights
  • Right rotate by k = last k elements go to front. [1,2,3,4,5] rotated right 2 = [4,5,1,2,3]
  • Always normalize: k = rotateBy % n before anything else
  • Three-reversal works: reverse left partition, right partition, then whole array
